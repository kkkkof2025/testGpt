演示如何使用 GPT5.2（Codex）直接连接 GitHub，完成在线编辑、测试、提交 PR 与 GitHub Pages 发布的全链路流程。适合受限设备、远程团队或临时需求。

连接 GitHub 并授权 查看文字版工作流

为什么选择在线协作？

零配置上手

无需本地 IDE，直接在浏览器或轻量终端完成修改、预览与提交。

分支与审阅友好

以 PR 为中心的流转：小步提交、自动检查、便捷回溯。

自动化驱动

结合 Actions / CI 工具，在云端跑测试、构建与部署，无需本地环境。

快速开始（Codex × GitHub）

1. 绑定与拉取
  • 在 Codex 授权 GitHub 访问权限。
  • 选择目标仓库并切换到新的工作分支（如 feature/online-edit）。
2. 在线编辑
  • 通过对话让 Codex 修改文件、生成补丁并查看 diff。
  • 将修改随时保存为提交，保持一次对话一次提交。
3. 验证与发布
  • 触发远程测试（Actions、Codespaces 或其他 CI）。
  • 创建 PR，附上测试输出、预览链接或截图，等待评审合并。

示例操作

示例 A：文档在线改写
  • 对话描述想要的章节结构，Codex 重写 readme。
  • 查看 diff、生成提交信息，推送到远程分支。
  • 在 PR 中说明修改目的和阅读指南。
示例 B：修复缺陷并跑测试
  • 定位 bug 代码，让 Codex 生成修复补丁并应用。
  • 在云端执行 npm test 或对应命令，将日志附到 PR。
  • 通过代码审阅后合并，自动部署到 Pages 或环境。
示例 C：更新 GitHub Pages
  • 在对话中指定要调整的模块（如本页文案/样式）。
  • 提交并推送，等待 Pages 构建完成后验证线上效果。
  • 在 PR 留下预览链接或截图，方便 Reviewer 审阅。

工作流清单

前置
  • 仓库已在 GitHub，必要的分支保护规则生效。
  • Codex 已获得 Git 权限，可读取/推送分支。
  • （可选）配置 Actions/CI 以支持自动测试与 Pages 部署。
执行
  • 通过对话驱动：描述需求 → 生成补丁 → 检查 diff。
  • 小步提交：每次修改后生成清晰的 commit 信息。
  • 必要时在远程环境运行测试或生成预览。
收尾
  • 创建 PR，填写变更摘要与测试结果。
  • 审阅通过后合并，并更新发布日志/标签。
  • 持续在 examples/ 积累 Prompt 与结果，方便复用。

适用角色

快速原型

产品/设计师可直接让 Codex 生成原型页面并推送到 Pages。

团队协作

开发与测试在同一 PR 下留下对话记录、日志与截图，降低沟通成本。

培训与演示

用最少准备展示「对话 → 修改 → 部署」的全流程，适合内部培训。
